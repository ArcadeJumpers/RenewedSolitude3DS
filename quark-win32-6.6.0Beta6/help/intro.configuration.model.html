 Mouse

 cdunde - 22 Jul 2008   [ Top ] 

To get the most out of the model-editor, you can specify here how the mouse should act when you do certain operations to it, like pressing a button and dragging.

It is highly recommended that you familiarize yourself with these mouse-settings, or redefine them if you like. But you should experiment with them in the model-editor, to get a feel of what they are doing.

However you should be aware that the mouse-modes default settings have been made to work with various elements of the editor and Skin-view and changing these settings could adversely effect how they work together.


 Options

 cdunde - 22 Jul 2008   [ Top ] 

Editor: 

A lot of things in the model-editor are controlled in this folder. A brief overview will be given here, but you should experiment with the settings yourself, to get a knowledge of what they do and if you can use them to something usefull.

Model Axis in views . This displays the models axis on which it was built in all views, showing its X, Y and Z direction. The size of its letter indicators and line thickness can be increased or decreased by using the 'Set Line Thickness' function. Their individual colors can be changed in the 'Configuration Model Colors' section.

Grid . Effects the display, visibility and activation of the editor's view grids only. The Skin-view has its own grid settings further below.
Cross-like cursor . If you're doing precision work, its better to have a cross instead of the arrow-shaped mouse-cursor.

There are numerous other items that you can set to customize the Model Editor to your choice.

Skin-view: 

Co-ordnances . This turns the function on or off to take co-ordnances for the Skin-view from the editors 3D view when new objects are created using the Quick Object Maker. This also applys when taking a Remap Snapshot It will place that object on the skin exactly the same way you see it in the 3D view when the object is created. This same process applies to selected faces in the editor. Because the 3D view uses its own scale factor, activating this function will also turn off the Use Skin-view scale function and visa-versa.

Grid . Effects the display, visibility and activation of the Skin-view grid only. The editor has its own grid settings further above.

Importers \ Exporters: 

This section applies strictly to the QuArK Model Editor's model import \ export logging system that is available. It is optional as to weather it is used or not by the creators of those files. Holding your mouse cursor over each item produces a Hint as to what its function is.

 Colors

 cdunde - 22 Jul 2008   [ Top ] 

So you don't like the default colors? Then change them here!

With various lines being drawn it can be very helpful to have contrasting colors.

The editor and the Skin-view have their own independent settings.

The same about contrasting colors apply even more so to the Skin-view since you will be dealing with textures having a whole array of colors.
